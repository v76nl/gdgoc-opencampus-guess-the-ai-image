# gdgoc-opencampus-guess-the-ai-image

AIによって生成された画像（ウォーターマークやExifなどの識別情報が埋め込まれた画像）と本物の画像を見分けるクイズアプリケーション、およびそのデータセットを生成するためのPythonスクリプトから成る体験型学習プロジェクト。

デモページ: https://v76nl.github.io/gdgoc-opencampus-guess-the-ai-image/

---

## 処理の流れ

### 1. データセットの準備 (prepare)
1. **画像の取得**: `download.py` を用いて、picsum.photosから一意なシードを用いて6枚の画像を取得します。
2. **ウォーターマークの埋め込み (AI画像タイプ1)**: `watermark.py` により、指定された画像のピクセルデータに対し、知覚しづらいノイズパターン（ウォーターマーク）を重畳します。
3. **Exifデータの埋め込み (AI画像タイプ2)**: `exif.py` により、指定された画像のExif領域（MakerNoteタグ）に、AI生成であることを示す文字列を埋め込みます。
4. **正誤表の自動生成**: `generate_answers.py` で、各画像の加工状態（Watermark/Exif/Real）を定義した `answers.txt` を出力します。

### 2. フロントエンドへの統合 (handson)
1. 生成された6枚の画像と `answers.txt` が `handson/public/images/` へ同期配置されます。
2. フロントエンド（ブラウザ）上で、画像がクイズカードとして並べて表示されます。

### 3. クイズ体験と解析シミュレーション
1. **クイズフェーズ (Phase 1)**: ユーザーは画像を視覚的に確認し、「AI生成」か「本物」かを推測して解答を選択します。
2. **検証フェーズ (Phase 2)**: 隠しターミナルに `answer` と入力することで、答え合わせダッシュボードへ遷移します。
3. **検証ツールの実行**: 各画像の「Exif解析」または「ピクセル解析」ボタンをクリックして `detector.ts` を呼び出します。
    - **Exif解析**: `exifr` ライブラリを使用してMakerNoteタグを読み取り、識別コードの有無を調べます。
    - **ピクセル解析**: Canvas経由でピクセルデータを読み取り、内蔵の透かしノイズとの相関係数を算出して判定します。
4. **開発者モード**: ターミナルで `debug` / `dev` / `admin` のいずれかを入力すると開発者モードになり、正解ラベルおよびAIの加工タイプ（ExifかWatermarkか）がダッシュボード上で可視化されます。ユーザーモード（`user` 入力時）では非表示になり、解析ツールを用いて答えを確認します。

## イベントの流れの構想
1. 運営側でアセットファイルを準備（リポジトリの状態で準備完了）します。
2. オープンキャンパス参加者にURLを共有し、ブラウザでクイズに回答してもらいます。
3. クイズ回答後、教員役が `answer` コマンドの使い方と裏側の判定技術（透かし・Exif）を解説し、答え合わせと解析のシミュレーションを体験してもらいます。

## 実行方法

### 1. データセット生成（Python環境）
`prepare` ディレクトリで以下のコマンドを実行します。

| コマンド | 実行内容 |
|---|---|
| `uv sync` | 依存関係パッケージの同期・インストール |
| `uv run python main.py` | アセット生成および同期パイプラインの実行 |

### 2. フロントエンド開発サーバーの起動（TypeScript環境）
`handson` ディレクトリへ移動し、以下のコマンドを実行します。

| コマンド | 実行内容 |
|---|---|
| `pnpm install` | 依存ライブラリのインストール |
| `pnpm dev` | 開発用ローカルサーバーの起動 |
| `pnpm build` | 本番用静的アセットのビルド |

起動完了後、`http://localhost:5173/` にブラウザでアクセスしてください。

## 画像枚数の変更とAI画像割合の仕様

### 1. 画像枚数の変更方法
`prepare/scripts/download.py` の `download_images()` 関数にあるダウンロードループ数を変更（例: `12` や `16` など）し、`prepare` ディレクトリで `uv run python main.py` を再実行します。
これにより、画像のクリア、新規ダウンロード、各処理、正誤表の生成、フロントエンドへのコピーが自動で行われます。

### 2. AI画像の割合ルール
全画像数 `N` に対して、ウォーターマーク（Watermark）とExifのAI画像割合は以下の計算式で決定されます：
- **ウォーターマーク（AI画像）**: $\lceil N / 4 \rceil$ 枚（小数切り上げ）
- **Exif（AI画像）**: $\lceil N / 4 \rceil$ 枚（小数切り上げ）
- **本物の画像（Real）**: 残りの枚数（$N - \lceil N / 4 \rceil \times 2$）

#### 構成例
| 総画像数 $N$ | ウォーターマーク (AI) | Exif (AI) | 本物の画像 (Real) | AI総数（割合） |
|---|---|---|---|---|
| **6枚** (現在) | 2枚 | 2枚 | 2枚 | 4枚 (約66.7%) |
| **8枚** | 2枚 | 2枚 | 4枚 | 4枚 (50.0%) |
| **12枚** | 3枚 | 3枚 | 6枚 | 6枚 (50.0%) |
| **16枚** | 4枚 | 4枚 | 8枚 | 8枚 (50.0%) |

フロントエンド側は `answers.txt` からアサイン情報を動的に解釈するため、フロントエンドのコードを修正することなく完全自動で同期されます。
